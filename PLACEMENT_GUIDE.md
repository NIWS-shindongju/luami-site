# 설치 사진/영상 제작 가이드 — 힉스필드 이미지·영상 작업 시 항상 적용

> 2026-07-06, 나랑한컷(onecut.co.kr) 설치사례 페이지 실사 분석 기반으로 작성.
> 앞으로 루아미 홈페이지/인스타그램용 이미지·영상을 힉스필드로 만들 때 이 문서를 먼저 참고할 것.

## ★ 작업 방식 (2026-07-06 갱신): 프롬프트 손글씨 대신 Reference Elements 사용

**매번 SCENE CONTEXT를 처음부터 길게 타이핑하지 않는다.** 힉스필드에는 재사용 가능한
Reference Elements(prop/environment)가 이미 있고, 매 세션 새로 만들 필요 없이 워크스페이스에
누적된다(`show_reference_elements(action:'list')`로 확인). 사용법:

1. **루아미 제품(prop) 요소** — 이미 등록됨: `luami-subway-hero`, `luami-elevator-hero`,
   `luami-premium-mirror`, `luami-case-kiosk`, `luami-season-red`, `luami-themezone`(환경).
   새 제품/테마가 생기면 실사 1장으로 `show_reference_elements(action:'create', category:'prop', ...)`.
2. **타사 설치 참조(environment) 요소** — "다른 업체 사진 학습" = 나랑한컷 등 실제 경쟁사 설치사례
   사진을 environment로 등록해두는 것. 이미 등록됨: `ref-corp-lobby`(기업 로비 단독설치),
   `ref-outdoor-festival`(야외 텐트+대기줄), `ref-luxury-mall-entrance`(고급 백화점 입구),
   `ref-brand-popup-colormatch`(브랜드 컬러 매칭 팝업). 설명에 항상 "REFERENCE ONLY — 타사
   실제 사례, 루아미 것 아님, 배치·조명 패턴만 참고, 로고/제품 디자인은 재현 금지"를 명시해둠.
   새로운 배치 유형(예: 웨딩홀, 골목 상가)이 필요하면 나랑한컷/유사 업체에서 실제 사진 1장을
   골라 같은 방식으로 environment 요소로 추가한 뒤 재사용한다.
3. **생성 시**: `generate_image`의 `prompt`에 `<<<prop_element_id>>>`와 `<<<environment_element_id>>>`를
   함께 넣고, 짧게 "이 prop을 이 environment의 배치 패턴(각도·간격·조명)에 맞춰 자연스럽게
   합성"만 지시한다. 모델(`nano_banana_2`/`nano_banana_pro` 등)이 두 참조 이미지의 실제 픽셀
   정보로 그림자·원근·색온도를 맞추므로, 텍스트로 일일이 서술하는 것보다 훨씬 안정적이고 빠르다.
   (텍스트 프롬프트만으로 처음부터 다 서술하는 방식은 이제 기본값이 아니라 예외 — environment
   레퍼런스가 없는 완전히 새로운 배치일 때만 쓴다.)
4. 결과물은 여전히 위 "핵심 원칙"·"체크리스트"·네거티브 프롬프트 규칙을 그대로 따른다 —
   Reference Elements는 입력 방식만 바꾼 것이지 품질 기준을 낮추는 게 아니다.

## ★ 카드뉴스(텍스트가 이미지에 박히는 콘텐츠)는 힉스필드 DTC Ads(`ms_image`) 사용 (2026-07-06 추가)

일반 이미지 모델(`nano_banana_pro` 등)에게 텍스트를 정확히 그리게 하는 시도는 반복 실패했다
(예: "VINTAGE ELEVATOR" 사이니지 3회 시도 모두 철자 깨짐). **텍스트가 들어가는 카드뉴스·광고
이미지는 처음부터 `ms_image`("DTC Ads")를 쓴다** — 이 모델은 텍스트 렌더링을 별도 파이프라인으로
처리해 100% 정확하게 나온다(검증: "왜 루아미인가" Us-vs-Them 카드, 오탈자 0, 브랜드 컬러 정확 적용).

사용법:
1. **제품 등록** (최초 1회, 제품별): `show_marketing_studio(action:'create', type:'product', medias:[...], title:..., description:...)` — 루아미 실사 1장이면 충분.
2. **브랜드킷**: 이미 등록됨 — `f3ea2494-4255-4eef-808c-7b3cf2662a5d` ("루아미 (Luami)", 컬러/톤/보이스 전부 세팅됨). 매번 `brand_kit_id`로 전달.
3. **스타일 선택**: `show_marketing_studio(type:'image_style')`로 42종 카탈로그 확인(Headline/Key Features/Benefits/Why We're Different/Product Spotlight/Customer Story 등) — 스토리 성격에 맞는 걸 매번 다르게 고른다(반복 금지, 다양성 요구사항).
4. **생성**: `generate_image({params:{model:'ms_image', style_id, brand_kit_id, product_ids:[...], prompt:'<이번 카피/USP 내용>', aspect_ratio:'3:4'}})`. aspect_ratio는 반드시 `1:1|3:2|2:3|16:9|9:16|4:3|3:4|21:9|27:16|16:27|9:8|8:9|4:9|9:4|auto` 중 하나(4:5 등은 거부됨).
5. 비용은 저렴함(1건당 약 0.5~2 크레딧) — 이미지당 여러 스타일 시도해보는 것도 부담 없음.
6. **주의**: "Why We're Different" 등 비교형 스타일은 내부적으로 "Us vs Them"(경쟁사 5약점 vs 루아미 5강점) 포맷으로 매핑됨 — 프롬프트에 특정 실제 경쟁사명을 넣지 말고 "타 렌탈 업체"처럼 일반화할 것(특정 업체 비방 금지 원칙 유지).

## ★ Canva 관련 참고 (2026-07-06)

이 계정은 Canva 무료 플랜이라 `search-brand-templates`(autofill 데이터셋)가 막혀 있다("Canva Pro 필요"
에러). `generate-design` AI 자동생성은 업로드한 자산/구체적 카피를 무시하고 엉뚱한 결과를 낸 전례가
있어 신뢰하지 않는다 — 카드뉴스는 위 DTC Ads 경로를 기본으로 쓰고, Canva는 Pro 전환 전까지 보류.

## 핵심 원칙: "타당함(plausibility)"이 사실감(realism)보다 먼저다

사진의 질감·조명·그레인이 진짜 폰카처럼 보여도, **왜 제품이 그 자리에 그 각도로 있는지** 설명이 안 되면
사람 눈에는 바로 "AI 티"로 읽힌다. 순서는 항상: **① 이 배치가 실제로 말이 되는가 → ② 그 다음에 사실적으로 렌더링.**
①을 생략하고 ②만 잘하면, 지금까지 실패한 사례들(하루필름 키오스크가 통로 한복판에 붕 뜬 것,
2대가 몸이 붙은 것)처럼 "품질은 좋은데 어색한" 결과가 나온다.

## 나랑한컷 사례에서 확인한 6가지 패턴

### 1. 키오스크(개방형, 벽걸이형)는 기본이 1대·단독
확인한 설치사례 대부분(KB증권, 시슬리 파리, 레디영 약국, 월악산, POSCO Park1538, 파리바게뜨 등)이
**단 1대**만 배치. 여러 대가 함께 있는 유일한 사례(동양생명 시상식, 카카오게임즈 4주년 페스티벌)는
- 각 기기가 확실히 분리된 **개별 공간/프레임**을 가지고
- 기기 사이에 **사람 한 명이 지나다닐 만한 실제 간격**이 있고
- 대기줄(로프 스탠션) 등 **다수 이용을 위한 동선 설계**가 함께 있는, 대형 행사/시상식용 특수 케이스뿐.
→ **일반적인 매장·팝업 키오스크는 무조건 1대 단독으로 제작한다.** 2대를 나란히/붙여서 만들지 않는다.

### 2. 배치 위치는 항상 "이유"가 캡션에 명시된다
- 시슬리 파리: "롯데백화점 1층 정문 바로 앞 — 방문객 시선을 자연스럽게 끌어들이는 자리"
- 레디영 약국: "2층 창가와 인접한 위치 — 개방감과 시인성 확보, 매장 내부로 유도"
- 한국지역자활센터협회: "슬림한 구조로 협소한 공간에도 쉽게 배치"
- 대한항공 라운지: 건축 니치(벽감)에 맞춤 제작해 통째로 붙박이처럼 통합
→ 이미지를 만들기 전에 **이 booth/kiosk가 왜 여기 있는지 한 문장으로 말할 수 있어야** 생성으로 넘어간다.
말이 안 되면(예: "그냥 넓은 통로 한가운데") 위치를 다시 정한다 — 입구 옆, 창가, 코너, 벽면 니치, 매장 진열대 옆 등.

### 3. 부스(밀폐형 캐빈)는 항상 벽·코너·기둥에 붙거나 건축적으로 통합된다
동선을 막는 위치는 단 한 건도 없음. 팝업스토어의 경우 부스 외관 색상/그래픽이 **매장 전체 컨셉과 통일**됨
(ERA 팝업 — 파란 벽에 파란 부스, 코인노래방 — 핑크 포토존 코너 전체를 핑크로).
→ 부스는 "벽에 붙었는가" + "그 매장/팝업의 톤과 어울리는가" 두 가지를 항상 확인.

### 4. 카메라 거리·각도: 제품 + 맥락이 둘 다 보이는 미디엄 샷, 3/4 각도가 기본
클로즈업으로 제품만 꽉 채우거나, 반대로 너무 멀어서 제품이 작아지는 사진은 거의 없음.
정면 각도보다 **살짝 비스듬한 3/4 각도**가 압도적으로 많음 — 입체감은 살리되 화면/브랜딩이 보이도록.
배경에는 항상 실제 위치를 증명하는 단서(에스컬레이터, 매장 쇼윈도, 행사 배너, 대기줄, 실내 인테리어)가 충분히 보이게 프레이밍.
→ 힉스필드 프롬프트에 "제품이 프레임의 40~55%만 채우고 나머지는 실제 맥락이 보이도록" 같은 구도 지시를 명시.

### 5. 사람이 자연스럽게 등장해도 됨 — 오히려 신뢰도를 높임
설치기사가 작업 중인 모습, 직원이 안내하는 모습, 고객이 사용해보는 모습 등이 그대로 포함됨.
너무 깨끗하게 비어있는 "쇼룸 컷"보다 **사람이 있는 "운영 중" 컷**이 더 진짜처럼 보임.

### 6. 야외/축제는 반드시 텐트·부스 구조물과 함께
야외 단독 설치는 없음 — 항상 브랜드 텐트, 배너, 파라솔 등 행사 인프라와 함께 배치됨.
"야외에 부스 하나만 덩그러니" 있는 컷은 나랑한컷 사례에 없음.

## 힉스필드 작업 시 체크리스트 (생성 전에 반드시 확인)

1. **이 제품(부스/키오스크)이 왜 이 자리에 있는지 한 문장으로 설명 가능한가?**
   (입구 옆이라 시인성이 좋다 / 창가라 개방감이 있다 / 코너라 동선을 막지 않는다 / 좁은 공간이라 슬림형이 필요했다)
2. **키오스크는 1대만 쓰는가?** (2대 이상은 특수 대형행사 케이스에만, 그것도 반드시 개별 프레임 + 실제 간격 + 동선 고려)
3. **부스는 벽/코너/기둥에 붙어 있는가, 아니면 그 공간의 건축 요소와 통합되는가?**
4. **카메라가 제품과 맥락을 동시에 보여주는 미디엄 샷·3/4 각도인가?**
5. **접지 그림자가 조명 방향과 일치하는가?** (붕 뜬 느낌 방지)
6. **가능하면 순수 텍스트 설명 대신, 이미 검증된 실제 배치 참조사진(reference image)을 두 번째 medias로 함께 제공한다.**
   (그림자·원근·색온도는 실제 픽셀 데이터에서 가져오는 게 텍스트 설명보다 훨씬 안정적 — Higgsfield/업계 공통 검증 사실.)
7. **카메라·렌즈를 물리적으로 명시**(예: 폰 후면 카메라 ~26mm, f/1.8)하고, 브랜드 텍스트는 따옴표로 정확히 지정하며,
   "3D render/CGI/게임엔진/일러스트/스튜디오 광택/시네마틱 색보정" 금지를 네거티브 프롬프트로 명시.

## 영상(Seedance 2.0) 리얼리즘 — 사람 동작이 핵심 트리거 (2026-07-06 추가)

이미지는 배치·각도만 잡으면 되지만, 영상은 **사람의 움직임과 카메라 안정성 자체가 "AI 티"의 최대 원인**이다.
정적인 겉모습이 아니라 움직임·표정의 미세한 부정확함(눈 깜빡임, 입모양 싱크, 손동작)에서 위화감을 느낀다는 게
2026년 기준 업계 공통 결론(uncanny valley 연구, Higgsfield Seedance 2.0 공식 프롬프트 가이드).

### 핵심 실수와 수정

1. **"고정 샷(static shot)"이라고 쓰면 오히려 부자연스럽다.** 실제 손으로 든 폰 영상은 완벽하게
   고정되지 않는다 — "handheld, 호흡·손떨림에 의한 미세한 지속적 흔들림, 완전히 고정되거나 안정화되지
   않음"을 명시. 단, 액션 신이 아니면 과도한 흔들림은 피하고 "차분한 정도"라고 톤을 지정.
2. **"자연스럽게 걷는다"만으로는 부족 — 물리(무게이동)를 직접 써준다.** 발뒤꿈치-발끝 착지, 걸음마다
   체중이동과 골반 흔들림, 팔이 다리와 반대로 스윙하는 반대회전, 머리카락·옷깃이 관성으로 움직였다가
   각 걸음 끝에 자연스럽게 정착하는 묘사 — "물리적으로 힘이 몸에 어떻게 작용하는가"를 서술해야 Seedance가
   질량·관성·결과를 만들어낸다(Higgsfield 공식 가이드 표현).
3. **손은 AI의 영원한 약점.** 손가락이 화면/물체에 정확히 닿는 정밀 동작은 왜곡되기 쉬우므로, 접촉
   직전에 컷을 끝내거나 손을 부분적으로 프레임 밖/블러 처리해 정밀 묘사를 피한다.
4. **초점·렌즈 결함을 일부러 넣는다.** "sharp but imperfect focus", 자연스러운 포커스 브리딩, 프레임
   가장자리의 미세한 색수차·비네팅 — 완벽한 클래리티 자체가 AI 티를 만든다.
5. **피부·텍스처는 명시적으로 "매끈함 금지".** "no smooth plastic/waxy skin", "visible pores and
   subtle imperfections", "no 3D render, no CGI, no cartoon, no VFX gloss"를 네거티브로 반드시 포함.
6. **샷 구조를 숫자로 명시.** "1 shot, N seconds, 이 화면비, no cuts, no zoom"을 맨 앞에 선언하고,
   행동을 "0-2초: ~, 2-4.5초: ~, 4.5-6초: ~" 식으로 시간 구간별로 쪼개 써준다 — 모호한 타이밍이
   어색한 동작의 원인이 된다.
7. **카메라 동사는 구체적으로.** "cinematic," "dynamic" 같은 막연한 단어 대신 dolly in / orbit /
   handheld / FPV 등 정확한 용어를 쓰고, 카메라가 "하지 않는 것"도 명시("no cuts, no zoom, natural
   head movement만")해서 시점을 고정한다.

## 영상 프롬프트 표준 템플릿 (2026-07-06, Higgsfield 최상급 프로덕션 분석 기반)

Higgsfield의 실제 최상급 프로덕션(Seedance 4K, "$350,000 AI Commercial" 브레이크다운) 프롬프트를
분석한 결과, 자유서술형 문단이 아니라 **섹션을 명확히 나눈 구조화된 프롬프트**를 쓴다는 게 핵심 차이였다.
앞으로 영상 제작 시 아래 템플릿을 기본으로 채워 넣는다 (해당 없는 섹션은 생략 가능, 순서는 유지).

```
SCENE CONTEXT: [한 문장 요약 — 누가 무엇을 왜 하는 장면인지]

ACTIVE REFERENCES: [기준 이미지의 제품/장소가 100% 일치해야 한다고 명시 — 디자인·로고·색상 등 구체적으로]

LOCATION MAP: [공간의 배치 — 무엇이 프레임 어느 쪽에 있는지, 통로/구조물 위치]

FIRST FRAME/BLOCKING: [영상 시작 프레임이 기준 이미지와 어떻게 일치하는지]

FORMAT MODE: [단일 연속 샷(no cuts, no zoom)인지 멀티샷(정확한 초 단위 컷 지점)인지, 총 길이, 화면비]

OPTICS: [정확한 화각(도 단위) 또는 환산 mm, 조리개값, 심도 — "와이드렌즈" 같은 막연한 표현 금지]

CAMERA: [handheld / 삼각대 고정 / 짐벌 등 구체적으로. 핸드헬드라면 "operator breathing, organic
  micro-shake, never locked-off" 같은 생생한 묘사]

ACTION (timed): [0:00-0:02, 0:02-0:04.5 등 초 단위 구간별로 동작을 쪼개서 서술 — 모호한 타이밍 금지.
  사람 동작은 "자연스럽게"가 아니라 무게중심 이동·보폭·팔의 반대회전 등 실제 물리량으로 서술]

PERFORMANCE: [감정·표정의 톤 — 과장 금지, 자연스러운 정도를 명시]

PHYSICS: [접지 그림자가 조명 방향과 일치, "nothing floats"를 반복 명시. 옷감·머리카락이 관성으로
  움직였다가 자연스럽게 정착]

LIGHTING: [색온도를 켈빈(K) 단위로 고정 — 기준 이미지와 색밸런스가 바뀌지 않도록]

STYLE: [자연스러운 센서 그레인, 초점 브리딩, 비네팅, 색수차 등 카메라 결함을 의도적으로 포함.
  "no 3D render/CGI/game-engine/과도한 피부 매끈함", "eyes stay natural, no eye glow"]

AUDIO: [diegetic만 쓸지 명시 — "NO MUSIC, SFX ONLY" 같은 강한 부정 표현이 배경음악 오남입을 막음]

POSITIVE LOCKS: [바뀌면 안 되는 것들을 마지막에 한 번 더 반복 명시 — 제품 디자인, 브랜드 텍스트, 장소]
```

### 사람 동작이 들어갈 때 특히 챙길 것
- 손가락이 화면/사물에 정확히 닿는 정밀 접촉은 피하고, 접촉 직전에 컷을 끝내거나 손을 부분적으로
  프레임 밖에 두어 AI의 최대 약점(손)을 정면으로 시험하지 않는다.
- 인물의 정면 얼굴·표정을 오래 클로즈업하지 않는다 — 측면/뒷모습으로 다가가는 구도가 훨씬 안정적이다.
- "고정 샷(static shot)"이라고 쓰지 않는다 — 실제 손 촬영은 완벽히 고정되지 않으므로 오히려 부자연스러움.

### 검증 결과 (2026-07-06)
이 템플릿 적용 후 하루필름 키오스크 영상에서: 걸음마다 무게중심 이동이 자연스럽고, 프레임 간 텔레포트/
워핑 없이 일관되게 이어지며, 손 접촉 직전 컷 + 측면 접근 구도로 AI 약점을 회피 — 이전 대비 뚜렷한 개선 확인.

  브레이크다운 — SCENE CONTEXT부터 POSITIVE LOCKS까지의 전체 섹션 구조, 캐릭터/프롭/장소 3-asset
  워크플로우, 정확한 화각(도 단위) 표기법, 색온도 켈빈 고정, 접지 그림자 반복 강조 기법

## ★ 컨셉 전환 (2026-07-06): 사람 등장 영상 폐기 → "제품 소개" 영상으로

키오스크 앞에 사람이 걸어와 사용하는 3편(v1~v3)까지 만들었고 v3는 리얼리즘 자체는 합격점(동주님 확인:
"영상의 현실감은 충분해")을 받았지만, **컨셉 자체가 기각**됐다: "장면 자체가 이야기가 없고 짧아 서사가
없어." 6초짜리 클립에 사람 1명 걸어와서 화면 만지는 동작만으로는 서사(왜 지금, 왜 이 사람, 그 다음은?)가
성립하지 않는다는 지적. → **앞으로 사람이 등장하는 영상은 만들지 않는다.** 제품(부스/키오스크) 자체가
주인공인 "제품 소개 영상"으로 전환.

### 사람 없이도 "서사"를 만드는 법 — 기능이 곧 스토리다

사람이 없으면 "감정 연기"로 서사를 만들 수 없다. 대신 **제품이 스스로 작동하는 상태 변화 순서**를
서사로 쓴다 — 이것 자체가 무인 포토부스의 핵심 가치("사람 없이도 알아서 찍고 알아서 인화한다")를
그대로 보여주는 것이기도 하다:

```
대기(idle) → 웨이크/활성화 → 핵심 기능이 실행되는 순간 → 완성된 결과물이 보이는 마무리 컷
```

루아미 제품(무인 포토부스/키오스크)에 대입하면:
1. **대기**: 화면이 슬립 상태(은은한 글로우만), 조용한 매장/공간에 단독으로 서 있음
2. **웨이크**: 화면 밝기 상승, 카메라 렌즈 링에 은은한 표시등, 안내 그래픽 활성화 — 사람이 다가온 걸
   감지한 센서 반응처럼 보이게(=실제 무인기기 동작 원리와 일치, 억지스럽지 않음)
3. **핵심 기능**: 인화구에서 사진이 실제로 밀려나오는 순간(즉석 인쇄), 또는 화면에 QR코드가 떠오르는
   순간(모바일 저장) — 루아미 실제 서비스 스펙("즉석 인쇄", "QR 저장")과 일치하는 장면만 사용
4. **마무리**: 카메라가 살짝 정착하며 완성된 3/4 히어로 컷 — 인화된 사진이 트레이에 살짝 보이는 채로 종료

→ 사람의 감정 연기 대신 "기기의 상태 변화"가 타임라인을 채운다. 6초 안에도 기승전결이 생긴다.

### 리서치 근거: 왜 이 구조인가

- **애플 제품 필름 분석**(diyphotography.net, manfrotto.com 등): 미니멀리즘·대칭·백라이트로 제품
  엣지를 분리하는 조명, 느린 리빌(reveal) 페이싱이 핵심. 단, **루아미는 스튜디오 5라이트 셋업을
  그대로 가져오면 안 된다** — 이 프로젝트의 대원칙("깨끗한 스튜디오 광고처럼 보이면 AI 티")과
  충돌하기 때문. → 조명 미학만 차용(엣지 분리감, 리빌 페이싱)하고, 촬영 방식은 계속 "실제 설치
  공간에서 폰카로 찍은 다큐 스타일"을 유지한다. 즉 카메라는 짐벌/스탠드에 놓고 천천히 미는 push-in ·
  약한 arc 정도만 쓰고, 360도 턴테이블 회전 같은 "광고티 나는" 카메라워크는 쓰지 않는다.
- **향수/명품 광고 프롬프트 리서치**: 제품이 공중에 뜨거나 안개·꽃잎이 흩날리는 연출은 화려하지만
  루아미 제품(사각형 무인기기)에는 안 맞고, 오히려 "AI가 만든 광고" 티가 강하게 남 → 사용하지 않음.
  접지·중력·질감은 항상 실사 기준을 유지.
- **Higgsfield 마케팅 스튜디오/캐릭터 사례**(HIGGS 브랜드 영상 브레이크다운): 색상 하나를 "서사의
  주인공"으로 고정해 제품을 계속 화면의 유일한 강조 요소로 만드는 기법(예: 형광색 의류만 컬러,
  주변은 무채색) — 루아미에도 적용 가능: 화면 UI 글로우·인화구 조명색을 일관되게 유지해서 시선을
  유도하는 장치로 사용.

### 사람 없는 영상의 네거티브 프롬프트 추가 항목
사람 동작 관련 항목(손 접촉 회피 등)은 이제 해당 없음. 대신:
- "no people, no human figures, no reflections showing a person, empty space" — 유리문·화면·거울
  반사에도 사람이 우연히 비치지 않도록 명시.
- 화면 UI 텍스트/그래픽은 AI의 새로운 약점(손을 대체하는 지점) — 화면에 뜨는 텍스트는 따옴표로 정확히
  지정하거나, 아예 "abstract glow, no legible text" 처럼 판독 불가능한 추상 그래픽으로 처리해
  깨진 텍스트 위험을 피한다.
- 인화되는 사진 용지의 물성을 구체적으로: "glossy photo paper, slight curl under gravity, motorized
  feed sound-free but visible mechanical motion, drop shadow matches ambient light direction."
- 비네팅은 이전 v3에서 "인스타 필터 같다"는 자체 피드백이 있었음 → **아주 약하게, 거의 안 보일
  정도로만** 지정한다. "extremely subtle corner falloff, NOT a stylized frame/oval vignette."
